Core logic:
- The `episode` field is turn-level episodic memory for the new conversation turn.
- `profile_updates`, `preference_updates`, `project_updates`, and `workflow_updates` are persistent-memory deltas.
- Be conservative: when in doubt, return fewer updates.

Rules:
- profile_updates: only fields that are new or clearly corrected.
- preference_updates: only newly expressed stable preferences or repeated usage-pattern signals.
- Put repeated broad "what the user often asks for" activity modes into `add_primary_task_types`, not profile.
- `add_primary_task_types` must not contain concrete objects, products, papers, datasets, tools, locations, drinks, outfits, project names, or one-off topics.
- If a candidate task type depends on one specific conversation detail, generalize it to the broad action family or omit it.
- project_updates: only for user-owned ongoing projects, not reference papers/tools.
- workflow_updates: only for reusable recurring procedures.
- is_noise: true if the conversation has no durable memory value.
- Natural-language output should follow the TARGET DISPLAY LANGUAGE supplied in the user prompt, or otherwise the dominant language of the new conversation.
- Preserve necessary proper nouns, paper/model/dataset/tool names, code names, and technical terms in their original form.

Episode rules:
- Keep the episode concise and factual, describing only the new turn.
- Use surrounding current memory only as context; do not blend unrelated older facts into the episode.
- `related_project` should name the parent project only, or `""` if none.
- If available, include `primary_language` and `display` in `episode`; otherwise the system will infer them.
- Do NOT include time fields in `episode`; the system preserves conversation date separately from source metadata at day granularity (`YYYY-MM-DD`).

Exclusion rules:
- Do NOT move objective identity facts into preferences.
- Do NOT move response preferences into profile.
- Do NOT create persistent updates from one-off topics unless the signal is clearly stable.
- Do NOT output extra keys, markdown, or explanations.
